# 貸し借りマップ - 要件と画面構成 v0

## プロジェクト方針
- 現行のサブスクボックス開発は中止し、この新規プロジェクトを優先する。
- まずは実装よりも、必要機能・画面構成・HTMLでのページ構成提案を最優先する。
- 今後の進行は、現行と同じく「小さいタスクを起こす → 実装/検証 → 完了後に次タスクへ進む」サイクルを継続する。

## アプリテーマ
他人との貸し借り状況を、金銭・ゲームソフト・本・ガジェット・日用品など対象を問わず可視化し、返却期限・リマインダー・履歴を管理するアプリ。

## コア価値
1. 誰に何を貸している/借りているかが一目で分かる。
2. 自分中心のマインドマップで、人とアイテムの関係を直感的に把握できる。
3. 期限・催促・返却済みの状態を忘れない。
4. 金銭だけでなく物品も同じUIで扱える。

## 主要機能
### MVP必須
- 貸し借り項目の追加
  - 種別: 貸した / 借りた
  - 相手
  - 品目名
  - カテゴリ: お金 / ゲーム / 本 / ガジェット / その他
  - 金額または数量/メモ
  - 貸借日
  - 返却予定日
  - リマインダー日時
- ホームの関係マップ表示
  - 中央に「自分」の丸
  - 相手は丸ノード
  - 貸し借り項目は四角いタグ状ノード
  - 貸した線と借りた線を2色で区別
- 一覧表示
  - すべて / 貸した / 借りた / 期限近い / 期限超過
- 詳細表示
  - 相手、品目、期限、メモ、履歴、リマインダー状態
- 返却完了処理
  - 完了日時とメモを残す
- リマインダー管理
  - 期限前・期限当日・期限超過の通知設定

### 次フェーズ候補
- 相手別プロフィール
- LINE/メール文面生成
- 写真添付: 貸した物の証拠/状態記録
- 一部返済・分割返済
- QR/リンク共有で相手に確認依頼
- 定期貸借/サブスク的な繰り返し
- 信頼スコアではなく「未返却傾向」程度の柔らかい可視化

## 画面構成案
### 1. ホーム: 貸し借りマップ
- 最重要画面。
- 中央に自分ノード。
- 周囲に相手ノード。
- 相手との間にタグ型の貸借アイテムを配置。
- 線色:
  - 貸した: オレンジ/赤系
  - 借りた: ブルー/シアン系
- タグ状態:
  - 通常
  - 期限近い
  - 期限超過
  - 返却済み
- 主要CTA: `+ 貸し借りを追加`

### 2. 追加/編集画面
- 最短入力を重視。
- 先に「貸した/借りた」を大きく選ぶ。
- 相手、品目、期限、リマインダーを入力。
- カテゴリはアイコン付きチップ。

### 3. 一覧画面
- カード/行形式で貸借項目を管理。
- フィルタ: すべて、貸した、借りた、期限近い、期限超過、返却済み。
- ソート: 期限近い順、作成順、相手別。

### 4. 相手別画面
- 特定の人との貸し借りだけを確認。
- 未返却件数、期限超過、履歴を表示。

### 5. リマインダー画面
- 今日/今週/期限超過の確認。
- 通知時刻の一括調整。

### 6. 履歴/アーカイブ
- 返却済みの記録。
- 後で「あれ返したっけ？」を確認できる。

## HTMLプロトタイプで表現する内容
- スマホアプリ想定の1ページHTML。
- 上部にアプリ概要と色凡例。
- 中央にマインドマップ風ホーム。
- 下部に画面タブ構成、追加フォーム、一覧カード、リマインダー案を並べる。
- 設計メモだけでなく、ユーザーがFBしやすいように実アプリ画面モックを必ず含める。
- 実アプリ画面モックは、追加・一覧・詳細・通知など主要導線をスマホ画面として見せる。
- アプリモックは別HTMLページとして作成し、概要ページURLとモックページURLを毎回の報告に併記する。
- モックページは静的な画面カード集ではなく、実際にアプリを起動した時の見た目と画面遷移/CTAのつながりをHTMLで再現する。
- まずは動作実装ではなく、ページ構成・情報設計・実画面イメージの提案を目的にする。

## 最初のDefinition of Done
- 新規フォルダが作成されている。
- 要件と画面構成がMarkdownで保存されている。
- HTMLプロトタイプが存在する。
- ローカルURLでHTMLを確認できる。
- 次に着手する小タスク案が用意されている。
